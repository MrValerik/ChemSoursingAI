# ChemSource AI — инструкция по запуску

Руководство для Windows. Технические детали (Docker, эндпоинты) — в docs/RUN.md.

## Что это

ChemSource AI — ассистент закупок химического сырья. Вы вводите параметры вещества,
система проверяет его по CAS-номеру, генерирует запрос поставщику (RFQ), извлекает
котировки из ответов и сводит их в сравнительную таблицу.

Приложение состоит из двух частей:
- бэкенд — логика и база данных (порт 8000);
- фронтенд — интерфейс в браузере (порт 5173).

## Шаг 1. Что установить (один раз)

1. Python 3.12+ — https://www.python.org/downloads/ (галочка "Add Python to PATH").
2. Node.js LTS (v22) — https://nodejs.org

После установки перезагрузите компьютер. Проверка:

    python --version
    node --version
    npm --version

Если npm ругается на "выполнение скриптов отключено" — выполните один раз и нажмите Y:

    Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

## Шаг 2. Установка зависимостей (один раз)

    cd D:\ChemSoursingAI\backend
    pip install -r requirements.txt

    cd D:\ChemSoursingAI\frontend
    npm install

## Шаг 3. Запуск (каждый раз)

Откройте два окна PowerShell.

Окно 1 — бэкенд:

    cd D:\ChemSoursingAI\backend
    python -m uvicorn app.main:app --reload

Дождитесь "Application startup complete".

Окно 2 — фронтенд:

    cd D:\ChemSoursingAI\frontend
    npm run dev

Откройте в браузере http://localhost:5173. Остановить — закройте оба окна.

## Шаг 4. Как пользоваться

1. Создайте запрос: введите CAS (например 50-78-2), наименование, базисы (CIP/FCA/EXW).
2. "Проверить CAS" — сверит вещество с базой PubChem.
3. "Предпросмотр письма" — покажет текст RFQ; "Создать RFQ" — сохранит запрос.
4. Вставьте ответ поставщика в блок "Ответ поставщика -> котировка", "Извлечь" распознает
   поля и уверенность; "Сохранить котировку" добавит её в сравнение.
5. Внизу — сводная таблица (полные котировки выше), фильтр "Только полные".
6. Нестандартные случаи попадают в очередь эскалаций для ручного разбора.

## Если что-то не работает

- npm не является командой: откройте новое окно PowerShell; не помогло — перезагрузите ПК.
- npm "выполнение сценариев отключено": Set-ExecutionPolicy -Scope CurrentUser RemoteSigned, затем Y.
- Бэкенд падает с ошибкой подключения к БД: нужен файл backend\.env со строкой DATABASE_URL=sqlite:///./dev.db
- Ошибка npm-prefix.js / битый node_modules: в frontend выполните cmd /c "rmdir /s /q node_modules", затем del package-lock.json и npm install.
- Браузер открыт, но данных нет: проверьте, что запущены оба окна (8000 и 5173).
- Порт занят: закройте старые окна с серверами и запустите заново.

## Демонстрация без браузера

    cd D:\ChemSoursingAI\backend
    python scripts/demo.py

Прогонит весь поток на тестовых данных и напечатает сводную таблицу и эскалации в консоли.
